# Harness Support

This document explains how to author skills, prompts, and MCP servers for different AI Harnesses, and how to verify they're working correctly in each supported surface: **Claude Code**, **GitHub Copilot**, **Codex**, and **Antigravity CLI**.

> **Note on Codex + Antigravity CLI:** Antigravity CLI (`agy`) is the successor to Gemini CLI and shares Codex's `.agents/` layout — skills at `.agents/skills/<name>/SKILL.md`, context at `.agents/context/AGENTS.md` (also `AGENTS.md` at the repo root), and MCP at `.agents/mcp_config.json`. They appear as a single target in the sync because they read the same files. See [the Antigravity migration guide](https://antigravity.google/docs/gcli-migration) for the upstream spec, and [the getting-started guide](https://antigravity.google/docs/cli-getting-started) for `agy` basics.

## Skills

A skill is a directory containing a `SKILL.md` file with `name` and `description` frontmatter. Optional subdirectories (`scripts/`, `references/`, `assets/`) can be added but are not required.

```

Use a slug-style `name` that matches the folder name, such as `code-reviewer` for `.agents/skills/code-reviewer/SKILL.md`. This keeps explicit invocation (`$code-reviewer`) and selector display consistent across Codex surfaces.
my-skill/
  SKILL.md        ← required: name + description frontmatter + instructions
  scripts/        ← optional
  references/     ← optional
```

**Description quality matters.** All skill descriptions are included in context so the agent can choose the right skill. Write descriptions that explain exactly when the skill *should* and *should not* trigger — front-load the key use case and trigger words. Codex caps the total skills list at ~8,000 characters and shortens descriptions when over budget, so brevity counts.

### Claude Code

Skills location (project): `.claude/skills/<name>/SKILL.md`
Skills location (user, all projects): `~/.claude/skills/<name>/SKILL.md`

Invoke: type `/` to list skills and slash commands.

Docs: https://docs.anthropic.com/en/docs/claude-code/skills

### Codex / Antigravity CLI

Skills location (repo root): `.agents/skills/<name>/SKILL.md`
Skills location (user, all repos): `~/.agents/skills/<name>/SKILL.md` (Codex) or `~/.gemini/antigravity-cli/skills/<name>/SKILL.md` (Antigravity global shared path)

Invoke: type `/skills` to list, or `$skill-name` to mention a skill directly. Both harnesses also support implicit invocation — if your task description matches a skill's description, the agent activates it automatically.

> **Note:** `.agents/skills/` is generated by the sync script and gitignored. Run `node .ai/scripts/sync.js` after cloning to create it.
> The target writes real generated `SKILL.md` files instead of file symlinks so Codex and Antigravity skill selectors can reliably discover them. Keep editing the canonical `.ai/skills/*.md` and `.ai/prompts/*.md` sources.

Docs: https://developers.openai.com/codex/skills (Codex) · https://antigravity.google/docs/gcli-migration (Antigravity skills layout) · https://antigravity.google/docs/cli-reference (Antigravity command reference)

### GitHub Copilot

Skills location (project): `.github/skills/<name>/SKILL.md`
Skills location (user, all projects): `~/.copilot/skills/<name>/SKILL.md`

Docs: https://docs.github.com/en/copilot/concepts/agents/about-agent-skills

## Prompts / Slash Commands

Prompts are slash commands — short instructions invoked explicitly via `/command-name`.

### Claude Code

Location: `.claude/skills/<name>/SKILL.md` (same directory as skills — `.claude/commands/` is deprecated)

Invoke: type `/` to list available skills and commands.

### GitHub Copilot

Location: `.github/prompts/<name>.prompt.md`

Docs: https://code.visualstudio.com/docs/agent-customization/prompt-files

### Codex / Antigravity CLI

There's no separate "prompt" or "custom command" file format — Codex and Antigravity treat prompts as skills. The sync script writes each `.ai/prompts/<name>.md` to `.agents/skills/<name>/SKILL.md` (the same target it uses for actual skills), so a prompt is invokable as `/name` or `$name`.

This unification matches Antigravity's behavior on first launch: `agy plugin import gemini` reports `commands : N legacy commands converted to skills` when migrating from Gemini CLI, confirming that commands no longer have their own file format.

## MCP

### Claude Code

- `.mcp.json` (project root, committed for team use)
- `.claude/mcp.json`

Docs: https://docs.anthropic.com/en/docs/claude-code/mcp

### GitHub Copilot

- `.mcp.json`
- `.vscode/mcp.json`
- `.github/mcp.json`

Docs: https://docs.github.com/en/copilot/how-tos/provide-context/use-mcp-in-your-ide/extend-copilot-chat-with-mcp

### Codex

Project-scoped MCP config via TOML (trusted projects only):

- `~/.codex/config.toml` — global (user-level) MCP servers
- `.codex/config.toml` — project-scoped MCP servers

Config format:
```toml
[mcp_servers.server-name]
command = "npx"
args = ["-y", "@foo/bar"]
env = { API_KEY = "value" }
```

The sync script generates `.codex/config.toml` from `.ai/mcp-servers.json`. Only the `command`, `args`, and `env` fields are carried into the TOML — any other per-server keys in the JSON source (e.g. `autoStart`) are dropped for Codex, though they pass through unchanged to the JSON targets (`.mcp.json`, `.agents/mcp_config.json`). It will not overwrite a manually edited file (ownership detected via `# ai-sync-generated` marker at the top).

Docs: https://developers.openai.com/codex/mcp

### Antigravity CLI

Antigravity separates MCP servers into a dedicated JSON profile (unlike Gemini CLI, which kept them inline inside `settings.json`).

- `~/.gemini/config/mcp_config.json` — global (user-level) servers
- `.agents/mcp_config.json` — project-scoped servers

Config format (same `mcpServers` shape as `.mcp.json`, but **remote-server URLs use the `serverUrl` key**, not Gemini's legacy `url`/`httpUrl`):

```json
{
  "_ai_sync_generated": true,
  "mcpServers": {
    "local-server": {
      "command": "npx",
      "args": ["-y", "@foo/bar"],
      "env": { "API_KEY": "$MY_API_TOKEN" }
    },
    "remote-indexer": {
      "serverUrl": "https://mcp.internal.enterprise.com/sse",
      "env": { "AUTH_TOKEN": "secure_alpha_token" }
    }
  }
}
```

The sync script generates `.agents/mcp_config.json` from `.ai/mcp-servers.json`, auto-renaming any legacy `url`/`httpUrl` keys to `serverUrl`. Ownership is detected via the top-level `"_ai_sync_generated": true` flag — a hand-edited file without that flag is preserved untouched. User-owned commands or per-server fields beyond the rename (env vars, args, command) pass through unchanged.

Inside a session, MCP tool calls appear with a confirmation prompt before invocation (standard MCP UX).

Docs: https://antigravity.google/docs/gcli-migration (MCP schema reference) · https://antigravity.google/docs/cli-reference (Antigravity command reference)

## CLI Commands Cheat Sheet

Each harness ships its own built-in slash commands for session control, context management, and tool discovery. The names overlap enough that muscle memory mostly transfers — this table maps the common actions across all four CLIs. Type `/` (or `?` in Copilot) inside any session to see the full live list; the per-tool docs at the bottom are authoritative.

| Action | Claude Code | Codex CLI | Antigravity CLI (`agy`) | Copilot CLI |
|---|---|---|---|---|
| Show / autocomplete commands | type `/` | type `/` | type `/` | `?` or type `/` |
| Clear / start fresh conversation | `/clear` | `/clear` or `/new` | see docs | (use `/compact`) |
| Compact context to free tokens | `/compact` | `/compact` | - | `/compact` |
| Show context / token usage | `/context` | `/status` | `/usage` | `/context` |
| Switch model | `/model` | `/model` | `/model` | (configured per agent) |
| Initialize project agent doc | `/init` (writes `CLAUDE.md`) | `/init` (writes `AGENTS.md`) | - | — |
| List / manage MCP servers | `/mcp` | `/mcp` | `/mcp` | `/mcp add` |
| List available skills | type `/` (skills appear) | `/skills` | `/skills` | `/agent` |
| Invoke a skill explicitly | `/skill-name` | `$skill-name` or `/skill-name` | `/skill-name` | `/skill-name` |
| Resume previous session | `/resume` | `/resume` | `/resume` | `/resume` |
| Submit feedback | `/feedback` | `/feedback` | see docs | `/feedback` |

### File and context mentions

Three of the four CLIs use `@` for inline file references, letting you attach a file's contents to the next message without quoting it manually.

| Mention | Claude Code | Codex CLI | Antigravity CLI | Copilot CLI |
|---|---|---|---|---|
| File reference | `@path/to/file` | `/mention` or `@path` | `@path` (Codex `.agents/` convention) | `@path/to/file` |
| Memory shortcut | `#` (prefix a message to append to `CLAUDE.md`) | — | — | — |
